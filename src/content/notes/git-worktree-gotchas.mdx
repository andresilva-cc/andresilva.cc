title: Git worktree gotchas
publishedAt: 2026-05-03
kind: til
---

Worktrees don't share `node_modules/` with the main checkout. If your linter or build tool runs from the local install, you need a fresh `pnpm install` inside the worktree before anything works. The error mode is misleading — you get "command not found" or "module not found" for tools that are obviously installed in the original repo.

Two more things I keep relearning:

- A worktree's branch is locked while the worktree exists. You can't check that branch out anywhere else until you remove the worktree or change its branch.
- `git worktree remove` refuses if the worktree has uncommitted changes. Add `--force` only after confirming the changes are actually disposable — otherwise stash or commit first.

Worth the friction. The isolation is the point.
