---
title: Type-safe env vars
publishedAt: 2026-05-09
kind: snippet
---

Validate environment variables at boot with Zod. Crashes fast on a missing or malformed value instead of producing a `undefined` somewhere three layers deep at request time.

```ts
import { z } from 'zod';

const envSchema = z.object({
  DATABASE_URL: z.string().url(),
  NODE_ENV: z.enum(['development', 'production', 'test']),
  PORT: z.coerce.number().int().positive().default(3000),
});

export const env = envSchema.parse(process.env);
```

The `z.coerce.number()` handles the fact that `process.env` is always strings. `.default(3000)` runs only when the key is genuinely absent — empty strings still fail the coercion, which is what you want.
