# Banned Buzzwords

A curated banned-list of clichés, empty marketing phrases, and résumé filler — with why each fails and what to write instead. The general writing craft behind these calls lives in `copy-craft.md`.

> **Toolkit-managed file — do not edit per-project.** These rules are static and shared across every project using the toolkit. Project-specific decisions belong in the project's own decisions doc — never here.

---

## Why this list exists

Buzzwords are not banned because they are ugly. They are banned because they carry no information. Every competitor claims to be innovative, robust, and seamless — so the words no longer distinguish anyone. They create the illusion of a claim while saying nothing checkable. The fix is always the same: replace the word with a concrete noun, a number, or a named action.

### Rule: A banned phrase is replaced, not deleted into vagueness
**Applies to:** Every entry below.
**Why:** Cutting "revolutionary" and leaving "our product" is not a fix. The replacement must add real information — what the product does, for whom, with what measurable result. If you have nothing concrete to put in its place, the problem is missing information, not word choice: ask the user for a specific fact.

### Rule: If a claim can't be substantiated, it can't be made
**Applies to:** Superlatives, "best-in-class", "world-class", "industry-leading", invented metrics.
**Why:** Unsubstantiated superlatives are both noise and a legal exposure. A reader who catches one empty boast discounts every other line. Make only claims you can back with evidence (see `copy-craft.md`).

### Rule: Adverb-and-adjective stacks are a buzzword smell
**Applies to:** Marketing headlines and feature copy.
**Why:** "Effortlessly powerful", "incredibly intuitive", "blazingly fast" stack two empty words and still say nothing. One concrete fact replaces the whole stack: not "blazingly fast" but "renders in 16 ms".

## Product and marketing copy

| Phrase | Why it fails | Use instead |
| --- | --- | --- |
| Revolutionary | Hyperbole the product can't earn; revolutions are rare and judged in hindsight, not announced. | State what changed: "the first X that runs without Y". |
| Game-changing / game-changer | Empty hype; names no game and no change. | Name the specific thing it removes or replaces. |
| Seamless | Has no antonym anyone would claim — nobody sells "seam-ful" software. Signals non-specificity. | Name the friction removed: "imports your CSV with no remapping". |
| Unleash | Software releases nothing; missionary energy with no mechanics. | Name the concrete capability and its result. |
| Supercharge | Energy metaphor with no mechanism. | Give the number: "cuts build time from 90 s to 12 s". |
| Next-generation / next-gen | Implies the predecessor was inadequate; offers no differentiation. | Name the specific new capability. |
| Cutting-edge / bleeding-edge | Claims novelty without content; ages badly. | Name the technology or the version. |
| State-of-the-art | Vague superlative; unverifiable. | Cite the benchmark or the standard met. |
| Innovative / innovation | Every product claims it, so it distinguishes nothing. | Show the innovation; let the reader conclude it. |
| Robust | Engineering-flavored filler; means "good" with no detail. | Give the fact: "handles 10k concurrent connections", "99.9% uptime SLA". |
| Powerful | Asserts capability without naming any. | Name the capability and a number. |
| Scalable | Used as a virtue with no figures; meaningless alone. | Give the range: "tested from 1 to 50M rows". |
| Empower / empower your team | Missionary language; dodges the concrete benefit. | Name what the user can now do that they couldn't. |
| Reimagine / rethink the way you... | Presumes the reader is dissatisfied before explaining the need. | State the concrete improvement directly. |
| Transform / transformative | Vague outcome; promises change without specifying it. | Name the before and after. |
| Elevate | Aspirational filler; names no concrete benefit. | State the measurable improvement. |
| Delight / delightful | Tells the reader how to feel instead of earning it. | Describe the moment; let delight be the reader's word. |
| Leverage (as a verb) | Inflated synonym for "use". | "Use." |
| Solutions | Abstract; hides what the product actually is. | Name the product: "a log parser", "an invoicing tool". ("Solutions" is the right word only when the company genuinely sells a multi-product portfolio.) |
| In today's digital landscape / in today's fast-paced world | Temporal hand-wave; assumes the reader needs orientation. | Delete it and open on the actual point. |
| Where X meets Y | False synthesis; creative laziness dressed as a tagline. | State plainly what the product is. |
| Built for the modern team | Vague; "modern" dates instantly and names no specifics. | Name the team: "for support teams handling 500+ tickets a day". |
| Best-in-class / world-class / industry-leading | Unverifiable superlative; every competitor says it. | Cite the ranking, benchmark, or award — or drop the claim. |
| Mission-critical | Borrowed gravitas; tells the reader nothing. | Name the consequence of failure. |
| Synergy / synergize | Corporate cliché; widely mocked, means "working together". | "Works with", or name the integration. |
| Disruptive / disrupt | Overused to meaninglessness; disruption is rare by definition. | Name what the product does differently. |
| One-stop shop | Tired idiom; vague about scope. | List what it actually covers. |
| Turnkey | Jargon; obscures what setup actually involves. | State the setup steps and time. |
| Frictionless | Same defect as "seamless"; no antonym. | Name the step removed. |
| Holistic | Vague; signals no real boundary on scope. | State what is and isn't included. |
| AI-powered | Used as a vague differentiator; everything claims it. | Name the model, the task, and the user-visible outcome ("uses GPT-4 to draft a reply you can edit"). |
| Magical | Tells the reader how to feel; hides the mechanism. | Describe what actually happens; let the reader call it magic. |
| Thoughtful | Self-congratulatory; unprovable. | Show the thought: name the decision and the tradeoff. |
| Beautifully crafted | Tells, doesn't show; aestheticizes without evidence. | Show the craft: a specific detail, a screenshot, a named material. |
| World-changing | Hyperbole; almost never earned. | Name the actual change to the user's day. |
| Ecosystem (as virtue) | Implies coverage without proving it. | Name the integrations and how many. |
| Platform (as virtue) | Used to inflate a product into a category. | Call it what it is — an app, a tool, an API — unless it literally hosts third-party developers. |

## Personal brand, résumé, and bio copy

| Phrase | Why it fails | Use instead |
| --- | --- | --- |
| Passionate developer / passionate about... | Everyone claims it; passion is told, not shown. | Show it: a side project, a contribution, a thing built unpaid. |
| Team player | Corporate filler; unprovable and universal. | Name the collaboration: "shipped X with a 4-person team across 3 time zones". |
| Hard-working / hardworking | Baseline expectation, not a differentiator. | Show the output, not the effort. |
| Results-driven / results-oriented | Empty buzzword; everyone's job is results. | Give the result: "grew signups 40% in two quarters". |
| Detail-oriented | Ironically often appears in résumés with typos. | Show it: "caught a billing bug that had over-charged 1,200 accounts". |
| Self-starter / proactive / go-getter | Vague self-praise; no evidence. | Name something you initiated and what came of it. |
| Fast learner | Asserts a trait; tells nothing. | Name a tool you picked up fast and what you shipped with it. |
| Rockstar / ninja / guru / wizard | Dated, unserious, and off-putting to most hiring readers. | State the actual role: "backend engineer", "infrastructure lead". |
| Motivated / driven | Filler; baseline expectation. | Show the motivation through a concrete pursuit. |
| Think outside the box | Cliché alert; the phrase is itself inside the box. | Describe the unconventional thing you actually did. |
| Strong communication skills | Asserted, not demonstrated; the bio itself is the evidence. | Let clear, specific writing prove it. |
| Dynamic / energetic | Vague personality filler. | Cut it; let accomplishments carry the energy. |
| Visionary / thought leader | Self-anointed; credibility must come from others. | State what you built or wrote; let readers judge. |
| Proven track record | Hollow without the record itself. | Show the track record: specific projects, dates, outcomes. |
| Wear many hats | Tired idiom for "did varied work". | List the actual roles. |
| Guru of / expert in (everything) | Diluted by overuse; expertise in all things reads as expertise in none. | Name a narrow, real specialty. |
