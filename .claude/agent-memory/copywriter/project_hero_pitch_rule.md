The home-page hero pitch (currently: "Software engineer with 9+ years of experience building web platforms, internal tools, and developer tooling.") must remain a single, self-contained statement sentence.

**Why:** Copy role drives column width on andresilva.cc — a short terminal "statement" line gets a narrow 56ch measure; multi-sentence "flowing prose" gets a wide 68ch measure. The hero pitch reads as a positioning statement (instant identity, absorbed in one glance). The Now-section paragraph is flowing prose at 68ch. The designer locked this role-based measure rule and flagged: the hero pitch only keeps 56ch while it stays one tight sentence.

**How to apply:** When editing the hero pitch, keep it to one sentence. If a future edit requires two-plus sentences of flowing copy, the block has changed role and must move from 56ch to the 68ch measure — flag this to the designer/orchestrator rather than silently widening.
